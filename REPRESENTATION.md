# Representation Design — Component Clauses and Bit Packing

Status: implemented through Stage 4 (see §5); two semantic rules were
discovered during implementation and are recorded in §3.4.
Scope: RM 13.1–13.4 record representation clauses, `pragma PACK` on
arrays, and every consequence they have downstream — sizing, `'SIZE`,
`'POSITION`/`'FIRST_BIT`/`'LAST_BIT`, element and component access,
aggregates, equality, and `UNCHECKED_CONVERSION`.

---

## 1. What GNAT does

GNAT implements the two features with **two disjoint mechanisms**.

### 1.1 Packed arrays: shadow implementation types (`Exp_Pakd`)

When a packed array type is frozen, GNAT manufactures a second type —
the `Packed_Array_Impl_Type`, named `xxxPn` — that describes the bits:

- **Constrained, static, ≤ 64 bits:** a modular integer subtype
  (`new Unsigned range 0 .. 2**nbits - 1`, or `Long_Long_Unsigned`).
  The array value *is* a scalar. On big-endian targets the value is
  stored left-justified so that the memory image matches the
  unconstrained representation.
- **Everything else:** a byte-array type `Packed_Bytes1/2/4`, the
  variant chosen by the component size's divisibility (sizes divisible
  by 2 or 4 get stronger alignment so the runtime can load wider
  windows).

Indexed references are rewritten during expansion into unchecked
conversions to the impl type. Component sizes 1, 2, 4, 8, 16, 32, 64
are expanded inline as shift/mask sequences; **every other size
(3, 5–7, 9–15, 17–31, …) calls a runtime library routine** — the
`System.Pack_NN` family, one generic get/set pair per bit width, over a
hundred runtime units, dispatched through the `Bits_Id` entity table.

Padding policy is split: the modular case **must keep unused bits
zero** (all operations maintain that invariant, so equality and
conversion are plain scalar ops); the byte-array case leaves tail bits
unspecified and clobberable.

Bit numbering: GNAT guarantees that bit 0 of a packed array coincides
with bit 0 of a record representation clause, so overlaying one on the
other via `UNCHECKED_CONVERSION` behaves as the clause reader expects.
On little-endian targets this is also the cheap order (subscript is
directly a shift count); on big-endian it costs reversal work
(`Rev_Packed_Bytes`, left-justification).

### 1.2 Record component clauses: delegated to the backend

`Sem_Ch13` validates clauses and normalizes each component to a
`Component_Bit_Offset` and `Esize`. The actual bit-field load/store
code is **not generated by the front end at all** — gigi maps clause-
placed components onto GCC's `DECL_BIT_FIELD` machinery and GCC emits
the shifts and masks.

### 1.3 GNAT-LLVM: the control experiment

GNAT-LLVM (`reference/gnat/gnatllvm-records-field_ref.adb`) is GNAT's
front end retargeted at LLVM — the same position we are in, without
GCC's bit-field machinery to lean on. What it built is instructive:

- A **`Bitfield_Ref_Desc`**: `{LHS pointer-or-data, field type, bit
  offset, bit size}`, with a `Normalize` step that folds whole bytes
  of offset into the pointer so the residual bit offset is always
  < 8 (`Byte' GEP + Offset mod BPU`).
- **`Build_Bitfield_Load`**: window size
  `Byte_Align (First_Bit + Num_Bits)`; load the window as an integer,
  `lshr` by the bit offset, `trunc` to the field width, then
  zext/sext by signedness (bitcast for float fields, inttoptr for
  access fields). `Build_Bitfield_Store` is the masked
  read-modify-write dual.
- **`Collect_Mixed_Bitfield`**: walks a nested reference chain
  (records inside arrays inside bit-placed components), accumulating
  a byte offset and a bit offset across the whole path before
  emitting one access — loci compose.
- One genuine edge case: a max-width field misaligned so its window
  exceeds the largest integer type by one byte is handled with two
  loads and an `llvm.fshr` funnel shift.

This validates the core emission strategy against a production
compiler. But GNAT-LLVM applies it **only to record components**:
packed arrays still arrive pre-expanded by `Exp_Pakd` through shadow
impl types and `System.Pack_NN` calls, because that expansion lives
upstream in the shared front end. Even with the right abstraction in
hand, GNAT-LLVM runs two mechanisms — the seam between front end and
backend is load-bearing and it cannot remove it. We have no such seam.

### 1.4 Costs of the GNAT shape

- Two parallel bit-access mechanisms that never share code: front-end
  expansion for packed arrays, backend bit-fields for records.
- A large runtime surface (the `Pack_NN` units) for the non-power-of-2
  sizes.
- Shadow types leak: they exist in the entity model, interact with
  freezing order, and expansion happens *during* analysis, which is
  why GNAT explicitly refuses to retype references ("this would cause
  all kinds of semantic problems").
- The design is constrained by the front-end/backend seam. GNAT-GCC
  delegates record bit-fields across it; GNAT-LLVM rebuilt them on its
  side of it; neither can unify record and packed-array access because
  `Exp_Pakd` runs before the seam. We have no seam.

## 2. What we do today (`ada83.c`) — and fail to do

### 2.1 What exists

- `Type_Info.size` is **bytes**; `specified_bit_size` records a
  bit-granular `'SIZE` when a clause supplies one.
- `Component_Info` carries `{byte_offset, bit_offset, bit_size}`.
  Default record layout assigns sequential byte offsets at natural
  size, `bit_offset = 0`, `bit_size = To_Bits (size)`.
- Record representation clauses are resolved by name match; `AT`
  (bytes) and `RANGE` (bits) land in those fields; the record's extent
  is computed bit-granularly and recorded in `specified_bit_size`.
- `'POSITION`, `'FIRST_BIT`, `'LAST_BIT`, `'SIZE` read the recorded
  layout — the *static* attribute story is right.
- Equality already knows representation coverage:
  `Equality_Needs_Componentwise` compares covered bits against
  `To_Bits (size)` and falls back to componentwise comparison when
  padding exists.
- `pragma PACK` sets `is_packed` on the array type.

### 2.2 What is broken or missing

1. **Layout is recorded but never obeyed.** No load or store consults
   `bit_offset`/`bit_size`. A component placed `AT 0 RANGE 0 .. 6`
   is still accessed as its full natural bytes at `byte_offset`.
   Sub-byte and overlapping layouts are silently wrong (cd4031a,
   cd4024b).
2. **`is_packed` is never consumed.** A packed boolean array occupies
   one byte per element; `UNCHECKED_CONVERSION` therefore sees the
   wrong sizes on both sides (cda201a, cd4024b).
3. **Element access is pointer-shaped.** The lvalue path returns a
   bare byte pointer, so a bit-granular target *cannot* be expressed —
   the bit displacement has nowhere to live. This is the structural
   reason packing was never wired in.
4. **Array byte-extent has no owner.** "length × element size" is
   hand-computed at ~6 sites (`Compute_Static_Array_Size`,
   `Static_Object_Byte_Size`, `Max_Constrained_Array_Size`,
   `Emit_Array_Byte_Size`, `Emit_Fat_To_Array_Memcpy`,
   `Emit_Fat_Pointer_Copy_To_Name`) with three different fallback
   conventions (1, 8, `FAT_PTR_ALLOC_SIZE`).
5. **Clause values are evaluated through `double`**
   (`Eval_Const_Numeric`), abandoning the compiler's own exact
   arithmetic (§6) and its > 2⁵³ guarantees.
6. Element access logic is **duplicated four ways** — rvalue index
   path, lvalue path, aggregate store, assignment element store —
   any representation change must be made identically in all four.

## 3. The design: one measure, one locus

Two principles, each strictly stronger than what either compiler has.

### 3.1 Bits are the only layout unit

All layout — component placement, element stride, type extent — is
measured in **bits**. Bytes are a boundary artifact of allocation and
block copies, derived at the edge via `To_Bytes` (ceiling). One
function owns extent:

```c
uint64_t Type_Bit_Extent (Type_Info *t);   /* every type, one answer   */
```

for arrays: `element_count × Array_Element_Bits (t)`, where
`Array_Element_Bits` returns 1 for a packed boolean array and
`To_Bits (element byte size)` otherwise; for records:
`specified_bit_size` if a clause spoke, else `To_Bits (size)`. The six
ad-hoc extent sites all become calls. GNAT mixes units across the
front end/backend seam; we stop mixing units at all.

### 3.2 The storage locus

Every readable or writable thing is named by a **locus**: a byte-
aligned base pointer plus a bit displacement and a bit width.

```c
typedef struct {
  uint32_t base_ptr;       /* reg: byte-aligned base address          */
  uint64_t bit_disp;       /* static displacement, bits               */
  uint32_t bit_disp_reg;   /* 0, or reg holding a dynamic displacement*/
  uint32_t width_bits;
} Storage_Locus;

Storage_Locus Locus_For_Component   (uint32_t base, Type_Info *rec, uint32_t comp_idx);
Storage_Locus Emit_Locus_For_Element(Type_Info *arr, uint32_t base, Aggregate_Operand flat);
LLVM_Value    Emit_Locus_Load  (Storage_Locus l, Type_Info *value_type);
void          Emit_Locus_Store (Storage_Locus l, LLVM_Value v, Type_Info *value_type);
```

Record components (defaulted or clause-placed), array elements (packed
or not) all *produce* a locus; exactly one load and one store
*consume* them. The four duplicated element-access sites and the raw
field load/store sites route through this pair.

**The fast path falls out instead of being special-cased:** when
`bit_disp % 8 == 0` and `width_bits` is a natural power-of-two width,
`Emit_Locus_Load` emits exactly today's `getelementptr i8 + load`.
Byte-granular code is the degenerate case of bit-granular code, so
the refactor is behavior-preserving by construction for every
existing program.

**The general path** (any displacement, any width ≤ 64):

```
first  = bit_disp >> 3                    ; first byte of the window
span   = To_Bytes ((bit_disp & 7) + width); window size, bytes
w      = load i(8·span), align 1          ; unaligned window load
v      = trunc (lshr w, bit_disp & 7) to iWidth
```

then zero/sign-extend by the component type's signedness. Store is the
dual read-modify-write: load window, `and` with the inverted mask,
`or` in the shifted value, store window. A dynamic displacement (an
indexed packed element) uses the same sequence with registers
(`lshr 3` / `and 7`). The window never exceeds the object: span is
derived from the field's own interval, and objects are allocated
`To_Bytes (Type_Bit_Extent)`.

This is, deliberately, the same sequence GNAT-LLVM's
`Build_Bitfield_Load`/`Build_Bitfield_Store` settled on (§1.3) — down
to the window formula — which is production validation of the
emission strategy. A misaligned maximum-width field can need a 9-byte
window; since we emit LLVM IR (not C), a plain `i72` unaligned load is
legal and LLVM legalizes it, so no funnel-shift special case is
needed — GNAT-LLVM only grows one for its C-emission mode.

**Loci compose.** A nested reference — a packed element of a record
component itself placed at a bit offset — is handled by *adding
displacements*: each step of the reference chain contributes
`byte-offset × 8 + bit-offset`, and one load/store is emitted at the
end (GNAT-LLVM's `Collect_Mixed_Bitfield` does the same walk). No
intermediate byte-materialization, no temporaries.

### 3.3 Deliberate policy decisions

- **No shadow types** (vs GNAT 1.1). We own the backend; LLVM's
  arbitrary-width integers and unaligned loads do inline what GNAT
  outsources to `Packed_Bytes` alignment tricks and 127 `Pack_NN`
  runtime units. Every width is the *same* four instructions. The
  entity model never learns packing exists.
- **No expansion-time rewriting** (vs GNAT 1.1). The syntax tree is
  never retyped; a locus is computed at emit time from `Type_Info`
  alone. GNAT's analysis-order contortions have no analogue here.
- **No zero-padding invariant** (vs GNAT's modular case). We never
  reinterpret a packed object as a scalar, so tail bits carry no
  obligation. Equality on a packed array with a partial tail byte
  compares whole bytes with `memcmp` and the tail byte under a mask —
  one helper, no invariant to maintain at every store.
  `Equality_Needs_Componentwise` already handles the record analogue.
- **Bit numbering: LSB-first within each byte** — flat bit *k* lives
  at byte `k >> 3`, bit `k & 7`. On our little-endian targets (§1
  target constants) this simultaneously satisfies both of GNAT's
  criteria: the subscript is directly a shift count, *and* bit 0 of a
  packed array coincides with `FIRST_BIT` 0 of an overlaid record
  clause, so `UNCHECKED_CONVERSION` between integers, packed arrays,
  and clause-placed records is representation-coherent. A big-endian
  port would revisit exactly one function pair.
- **Packing chooses 1 bit only for two-valued types.** RM 13.1 leaves
  the packing degree to the implementation; for boolean (and any
  2-literal enumeration) we pack to 1 bit; other component types keep
  their natural size under `pragma PACK`. Widths that would straddle
  usefully (2, 4-bit) can be added later inside the locus layer
  without touching any call site — that is the point of the layer.
- **Addressability is the honest cost of packing.** A locus with
  `bit_disp % 8 != 0` has no address: `'ADDRESS` on such a component
  is diagnosed, and parameter passing uses copy-in/copy-out (Ada 83
  permits this for packed components; independence of addresses is
  exactly what `pragma PACK` trades away).
- **Clause values are folded with the compiler's exact arithmetic**
  (§6 big integers), not `double`.

### 3.4 Semantic rules discovered during implementation

Two rules were forced by ACATS and by address-sensitive flakiness, and
belong to the design:

- **Padding canonicalization at conversion boundaries.** A bit-packed
  view owns only its value bits. `UNCHECKED_CONVERSION` *into* a
  packed view zeroes the final partial byte's padding bits (the
  buffer's tail), and stores never touch bits outside their interval —
  so padding read back later is deterministic. Without this, the
  padding carries whatever the stack held, and equality through
  scalar reinterpretation becomes ASLR-dependent (observed as a
  50%-flaky roundtrip before the rule was adopted).
- **Scalar extraction extends from the packed width.** Converting a
  packed image of N bits to a scalar of B > N storage bits yields the
  low N bits extended by the *destination's* signedness: a 7-bit
  `-63` is `0x41`, and only an arithmetic shift pair restores `0xC1`.
  This is the semantic the ACATS `LENGTH_CHECK` roundtrips assume,
  and it matches what GNAT gets from converting through its N-bit
  modular impl type.

Also adopted: general element widths (a discrete element with a 'SIZE
clause smaller than its natural size packs to that width, straddling
byte boundaries), `pragma PACK` record layout (discrete components
laid end to end at bit granularity, composite ones byte-aligned), and
allocation padding of `To_Bytes(7 + W) - 1` bytes for straddling
strides so runtime windows never leave the object — the same job
GNAT's `Packed_Bytes2/4` alignment does.

## 4. Why this beats all three

| Concern | GNAT-GCC | GNAT-LLVM | ada83.c today | This design |
|---|---|---|---|---|
| Mechanisms for packed arrays vs record clauses | two, disjoint | two (bitfield desc + inherited `Exp_Pakd`) | two, both inert | **one** (locus) |
| Non-power-of-2 widths | runtime library calls | inline for records, `Pack_NN` calls for arrays | unsupported | inline, one code path |
| Shadow/impl types | yes, leak into entities | inherited for arrays | none | none |
| Tree rewriting during analysis | yes | inherited | none | none |
| Padding-bit invariant | required (modular case) | inherited | n/a | not required |
| Extent computation | split front/back end | split | 6 ad-hoc sites | one function |
| Element access sites | expander rewrites | expander + bitfield desc | 4 duplicates | 1 producer/consumer pair |
| Fast path for unpacked code | separate code | separate code | is the only code | degenerate case of general path |

## 5. Migration plan (staged, each stage compiles, sweeps, commits)

1. **Extent unification.** Introduce `Array_Element_Bits` /
   `Type_Bit_Extent`; convert the six extent sites. Pure refactor —
   bit-identical IR for all existing inputs.
2. **Locus layer over records.** Introduce `Storage_Locus` and the
   load/store pair; route `Emit_Record_Field_Ptr` consumers through
   it. Honor `bit_offset`/`bit_size` from component clauses.
   Unblocks: cd4031a, half of cd4024b.
3. **Packed arrays, static bounds.** `Array_Element_Bits` = 1 for
   packed booleans; route the four element-access sites through
   `Emit_Locus_For_Element`; aggregate zero-fill; masked-tail
   equality; `UNCHECKED_CONVERSION` sizes follow `Type_Bit_Extent`.
   Unblocks: cda201a.
4. **Packed arrays, dynamic bounds.** Runtime locus displacement;
   arrays of packed arrays (`To_Bytes` stride).
   Unblocks: cd4024b (`LENGTH_CHECK`'s `ARRAY (1 .. LEN)`).

Landing constraint: full Class C sweep against `acats.baseline` at
every stage; zero regressions.
